## 📝 Definition of Ready (DoR)

Before starting a task, ensure:
- [ ] Task has clear acceptance criteria
- [ ] Task has story point estimate
- [ ] Dependencies are identified
- [ ] Technical approach is understood
- [ ] Required resources/access available
- [ ] Task is assigned to a developer

---

## ✅ Definition of Done (DoD)

Before marking a task complete:
- [ ] Code implemented per acceptance criteria
- [ ] Unit tests written (80%+ coverage)
- [ ] Integration tests written (if applicable)
- [ ] Code review completed and approved
- [ ] No critical/high severity bugs
- [ ] Documentation updated
- [ ] Deployed to staging environment
- [ ] Manual testing completed
- [ ] QA sign-off received
- [ ] Ready for production deployment

## 🎬 Sprint Ceremonies

### Daily Standup
- **Time**: Every day at 10:00 AM
- **Duration**: 15 minutes
- **Format**: Each person answers:
  1. What did I complete yesterday?
  2. What will I work on today?
  3. Any blockers or impediments?

### Sprint Review
- **Time**: [Last day of sprint] at 3:00 PM
- **Duration**: 1 hour
- **Attendees**: Team + Stakeholders
- **Agenda**:
  1. Demo completed features
  2. Review sprint goal achievement
  3. Gather feedback
  4. Update product backlog

### Sprint Retrospective
- **Time**: [Last day of sprint] at 4:00 PM
- **Duration**: 1 hour
- **Attendees**: Team only
- **Format**:
  1. What went well?
  2. What didn't go well?
  3. What will we improve next sprint?
