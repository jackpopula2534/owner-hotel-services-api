ปัญหาของระบบปัจจุบัน (Current Pain Points)

❌ ไม่มี Template — KPI Hardcoded

น้ำหนัก KPI ถูก hardcode ทั้งใน Frontend (5 ตัว) และ Backend (4 ตัว) แต่ไม่ตรงกัน ทำให้ user ไม่รู้ว่าควรกรอกอะไร และไม่สามารถ customize KPI ตามแผนก/ตำแหน่งได้

✅ แก้: สร้าง KPI Template ที่ configurable ได้

❌ ไม่มี Evaluation Cycle — ต้อง Add ทีละคน

ปัจจุบัน user ต้องกด "+ เพิ่มการประเมิน" แล้วเลือกพนักงานทีละคน ทำให้ถ้ามี 50 คน ต้องทำ 50 ครั้ง ไม่มีระบบ batch generation ให้

✅ แก้: เปิด "รอบประเมิน" ครั้งเดียว — auto-generate ให้ทุกคน

❌ Slider ใช้งานยาก และ งง

ใช้ slider (0-100) โดยไม่มี guidance ว่าแต่ละ score หมายถึงอะไร ผู้ใช้ไม่รู้ว่า 70 คือ "ดี" หรือ "แย่" ทำให้คะแนนไม่สอดคล้องกันระหว่างผู้ประเมิน

✅ แก้: ใช้ rating scale พร้อม label (1-5 หรือ 1-10 ที่มีคำอธิบาย)

❌ "Invalid Date" + "undefined" ใน List

รายการ performance แสดง "Invalid Date" และ "undefined" เพราะ period format ไม่ตรงกับที่ backend ส่งมา และ department field map ไม่ถูก

✅ แก้: fix date format + type mapping ระหว่าง frontend-backend

❌ ไม่มีขั้นตอน Approval ที่ชัดเจน

ปัจจุบัน status มีทั้ง draft/submitted/approved/rejected แต่ไม่มี UI workflow ที่ชัดเจนว่า "ใครทำอะไรในขั้นตอนไหน" manager จะ approve ได้อย่างไร?

✅ แก้: สร้าง approval workflow พร้อม role-based action

❌ ไม่มี KPI Summary / Report

ไม่มีหน้าสรุปผลที่แสดง grade distribution, แนวโน้ม, top/bottom performers ทำให้ผู้จัดการไม่สามารถใช้ข้อมูลนี้ในการบริหารได้จริง

✅ แก้: เพิ่ม Dashboard summary + Export PDF/Excel

เปรียบเทียบ: ก่อน vs หลัง Redesign
จุด ❌ ระบบเดิม (ปัญหา) ✅ ระบบใหม่ (แก้แล้ว)
จุดเริ่มต้น กด "+ เพิ่มการประเมิน" แล้วกรอกทีละคน ทีละรอบ ตั้ง Template → เปิดรอบ → ระบบ auto-generate ให้ทุกคน
KPI Criteria Hardcode 4-5 รายการ ปรับไม่ได้ frontend-backend ไม่ตรงกัน Configurable ผ่าน Template น้ำหนักรวม validated = 100%
Input Method Slider (0-100) ไม่มี guidance ใช้งานยาก Rating 1-5 พร้อม label คำอธิบายแต่ละระดับ
Approval ไม่มี workflow ชัดเจน status เปลี่ยนได้เองโดยไม่มี role control Role-based: HR กรอก → Submit → Manager Approve/Reject
Invalid Date bug "Invalid Date" และ "undefined" ใน list Fix format mapping + ใช้ cycle period แทน manual period string
Batch Operations ต้อง add ทีละคน ถ้า 50 คน ทำ 50 ครั้ง เปิด Cycle ครั้งเดียว auto-generate 50 records พร้อมกัน
Reports ไม่มี summary, ดูได้แค่ list รายคน Grade distribution, Top performers, Export PDF/Excel

สรุปสิ่งที่ต้องสร้าง (Implementation Roadmap)

📌 ตอบคำถาม: "ต้อง Set Template ก่อนไหม?"

ใช่ครับ — ลำดับที่ถูกต้องคือ Template ก่อน → เปิด Cycle → กรอกคะแนน → Approve → ดูผล
Template ทำครั้งเดียว แต่ Cycle เปิดทุกรอบ (Q, ครึ่งปี, ปี) ไม่ต้องกรอก KPI criteria ซ้ำทุกครั้ง

1

Backend: สร้าง Prisma models ใหม่ — KpiTemplate, KpiCriteria, EvaluationCycle, KpiScore + migrate HrPerformance

2

Backend: สร้าง NestJS modules: kpi-template, evaluation-cycle พร้อม endpoint สำหรับ auto-generate performance records

3

Frontend: สร้าง UI "ตั้งค่า KPI Template" — form + drag-reorder criteria + weight validator (รวม = 100%)

4

Frontend: Redesign หน้า "ผลการปฏิบัติงาน" — เปลี่ยน flow เป็น Cycle-based, ใช้ Rating 1-5 แทน Slider

5

Frontend: สร้างหน้า "รออนุมัติ" สำหรับ Manager พร้อม Bulk Approve + Comment on Reject

6

Frontend: สร้างหน้า "สรุปผล" — Grade chart, Top performers, Export PDF/Excel
